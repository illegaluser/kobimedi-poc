## 1단계: 대화 품질 기본기 다지기 (2026-03-25)

챗봇과 실제로 대화해보니, 가장 기본적인 대화 흐름에서 세 가지 문제가 있었습니다.

### 해결한 문제

| 문제 | 설명 | 해결 |
|------|------|------|
| 본인/대리 예약 확인 스킵 | "예약해주세요"라고 하면 본인 예약인지 묻지 않고 바로 진행 | 예약 시작 시 반드시 "본인이신가요?" 질문이 먼저 나오도록 수정 |
| 이름+연락처 동시 입력 인식 실패 | "김민수 010-1234-5678"처럼 한 번에 입력하면 인식 못 함 | 한 문장에서 이름과 전화번호를 동시에 추출하도록 개선 |
| 불필요한 상담원 연결 | 정보를 순서대로 잘 알려주고 있는데도 "상담원 연결합니다"로 전환 | 사용자가 정보를 제공하며 대화가 진전되는 중이면 상담원 연결 방지 |

### 응답 문구 개선

상황별로 챗봇이 다르게 질문하도록 개선했습니다.

- **예약/변경/취소/조회** 각각에 맞는 본인확인 질문 문구 적용
- 이름과 연락처가 모두 없을 때 "성함과 연락처를 함께 알려주세요"처럼 한 번에 요청

**품질 검증**: 대화 시나리오 테스트 13건 전체 통과, 응답 문구 테스트 27건 전체 통과

**품질 검증**: 자동 평가 정확도 32% → **62%** (+30%p 향상)

## 10단계: 환자 이름 오추출 버그 수정 (2026-03-26)

### 배경
9단계의 E2E 테스트에서 "내과 예약하고 싶어요"라는 첫 메시지에서 **"싶어"가 환자 이름으로 잘못 추출**되어 cal.com 예약의 참석자 이름이 "싶어"로 기록되는 문제를 발견했습니다.

### 원인
이름 추출 로직이 문장의 각 단어에서 접미사 "요"를 제거한 뒤, 남은 2~3글자 한글을 이름 후보로 판정합니다. "싶어요"에서 "요"를 제거하면 "싶어"(2글자 한글)가 남아 이름으로 오인식됩니다.

같은 방식으로 "좋아요"→"좋아", "해줘요"→"해줘", "볼게요"→"볼게" 등 거의 모든 동사/형용사 표현이 이름으로 오추출될 수 있었습니다.

### 해결
접미사를 제거하기 **이전에** 동사/형용사 활용형 토큰을 먼저 식별하여 이름 후보에서 제외하는 가드를 추가했습니다.

| 입력 예시 | 수정 전 | 수정 후 |
|-----------|---------|---------|
| "예약하고 싶어요" | "싶어" (이름으로 오인식) | None (동사 활용형 → 스킵) |
| "확인 좀 해줘요" | "해줘" (이름으로 오인식) | None (동사 활용형 → 스킵) |
| "김민수 010-1234-5678" | "김민수" | "김민수" (정상 유지) |

**품질 검증**: 자동 테스트 163건 전체 통과, 전체 평가 51/51 (100%)
