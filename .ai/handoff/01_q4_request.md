# Q4: cal.com 연동 요구사항 (Request)

## 1. 개요
코비메디 AI 에이전트의 선택 과제인 Q4 구현을 위한 요구사항 정의서입니다. 에이전트가 로컬 환경에서의 정책(Policy) 및 저장소(Storage) 검증을 통과하여 `book_appointment`를 확정하기 직전에, 실제 외부 예약 시스템인 Cal.com과 연동하여 이벤트를 생성합니다.

## 2. 세부 요구사항
1. **이벤트 타입 설정**
   - `ent-consultation` (이비인후과, 30분)
   - `internal-medicine` (내과, 30분)
   - `orthopedics` (정형외과, 30분)

2. **예약 파이프라인 연동**
   - 로컬 정책 검증 이후, 예약 확정 전에 cal.com의 **Available Slots API (v2)** 로 가용 시간을 조회합니다.
   - 이미 외부에서 예약이 차서 가용하지 않은 경우, 로컬 예약 확정을 중단하고 고객에게 가용 시간을 공유하는 응답(`clarify`)을 생성합니다.
   - 사용자 확인이 완료되면, **실제 Booking 생성 API (v2)** 를 호출하여 cal.com 캘린더에 이벤트를 반영합니다.

3. **정책의 자연스러운 결합**
   - 코비메디 로컬 정책(1시간당 3명 제한, 운영시간 등)을 우선 적용하며, cal.com의 슬롯 가용성은 최종 방어선 및 동기화 수단으로 작용합니다.

4. **거짓 성공 금지 (Fail-safe)**
   - cal.com API 장애, 타임아웃, 토큰 오류 등이 발생할 경우 절대 고객에게 "예약이 완료되었다"고 응답해서는 안 됩니다 (`KpiEvent.AGENT_HARD_FAIL` 기록 및 안전한 폴백).

5. **증빙 자료**
   - 구현 완료 후 cal.com 캘린더 대시보드에서 생성된 예약 스크린샷 1-2장을 산출물로 제출합니다.
