# Q4: cal.com 연동 정책 (Policy)

## 1. 아키텍처 및 통합 위치
cal.com 연동은 시스템의 진실원천(Source of Truth)인 **로컬 파일 저장소(`bookings.json`)의 보조 수단**으로 동작합니다. 
외부 연동 로직은 반드시 `src/calcom_client.py` 단일 진입점에 격리하며, `src/agent.py`에서만 호출합니다.

- **호출 시점 1 (Slot 가용성 확인)**: 멀티턴 대화 중 사용자가 날짜와 시간을 모두 제공하여 예약을 확정하는 확인 질문(Confirmation)을 던지기 직전.
- **호출 시점 2 (Booking 생성)**: 사용자가 최종적으로 예약을 승인(`네`, `진행해주세요` 등)하여 로컬 저장소에 `create_booking`을 수행하기 직전 (또는 배치 모드에서의 확정 직전).

## 2. Cal.com v2 API 스펙 준수
- **인증**: `Authorization: Bearer <API_KEY>` 형식 사용.
- **버전 헤더**: 엔드포인트별 필수 헤더인 `cal-api-version`을 반드시 포함 (예: slots 조회는 `2024-09-04`, bookings 생성은 `2024-08-13`).
- **데이터 포맷**: 예약 시간 및 가용 슬롯 요청/응답은 ISO 8601 포맷(`YYYY-MM-DDTHH:MM:SS+09:00`)을 엄격히 준수합니다.

## 3. 거짓 성공 금지 (Fail-safe) 및 예외 처리 정책
| 장애 상황 | 시스템 대응 (Action) | 고객 응답 예시 | KPI |
| --- | --- | --- | --- |
| API Key 미설정 | 연동 로직 우회 (Graceful Degradation) | 정상 로컬 예약 처리 진행 | - |
| Slot 조회 Timeout | `clarify` 폴백 | "현재 외부 예약 시스템 응답 지연으로 가용 시간을 확인할 수 없습니다." | `AGENT_HARD_FAIL` |
| 외부 Slot 마감 | `clarify` 폴백 + 대안 제시 | "선택하신 시간은 방금 마감되었습니다. 가능한 다른 시간은..." | `AGENT_SOFT_FAIL_CLARIFY` |
| Booking 생성 실패 | `clarify` 폴백 | "예약 시스템 응답 지연으로 예약이 확정되지 않았습니다. 다시 시도해주세요." | `AGENT_HARD_FAIL` |

## 4. 데이터 매핑 정책
- **환자 식별**: Cal.com은 이메일을 필수로 요구하는 경우가 많으므로, 이메일이 없는 전화 예약 시스템 특성상 환자 연락처를 활용한 더미 이메일(`01012345678@kobimedi.local`)을 생성하여 전달합니다.
- **메타데이터**: 실제 환자 연락처는 예약자 노트(`notes` 또는 `metadata`) 필드에 명시적으로 삽입하여 의료진이 대시보드에서 확인할 수 있게 합니다.

## 5. 기존 정책과의 충돌 해결
- 코비메디 로컬 정책(초진 40분, 재진 30분, 1시간 정원 3명)은 `src/policy.py`에서 **먼저 검증**합니다.
- 로컬 정책을 통과하더라도 외부 Cal.com 캘린더에서 이미 슬롯이 점유되었다면 외부 상태를 최종적으로 존중합니다.
