# Q1: 코비메디 AI Agent PoC 성공 지표 (Metric Rubric) 제안서

## 1. 비용 구조 분석에 따른 최적화 방향

코비메디의 예약 처리 과정에서 발생하는 비용 구조를 분석해보면, 이번 PoC가 나아가야 할 방향이 명확해집니다.
- **AI 성공 처리 (Agent Success)**: 건당 +$10 절감
- **AI 실패 → 상담원 연결 (Agent Soft Fail)**: 건당 -$20 비용 발생
- **AI 실패 → 고객 이탈 (Agent Hard Fail)**: 건당 **-$500 손실** 발생

가장 눈여겨봐야 할 부분은 **'하드 실패(고객 이탈)'의 치명적인 영향**입니다. 단 1건의 하드 실패만 발생해도, AI가 50건의 예약을 성공적으로 처리해 얻은 절감액(+$500)이 모두 날아가 버립니다. 

즉, 눈에 보이는 자동화율을 높이겠다고 불확실한 예약을 억지로 확정하거나, 챗봇이 의학적 근거 없이 의료 상담을 진행하는 것은 비즈니스 관점에서 최악의 결과를 초래합니다. 

따라서 본 AI Agent는 시스템 아키텍처 레벨에서 다음 세 가지 차별화된 원칙을 적용하여 설계되었습니다.
1. **치명적 실패(-$500) 원천 차단 (결정론적 정책 엔진 & 본인/대리인 선확인)**: LLM의 환각을 배제하기 위해 예약 정원, 시간 계산 등을 파이썬 코드(Rule Engine)로 강제했으며, 환자 식별 오류를 막기 위해 대화 첫 턴에 '본인/대리인' 여부와 '전화번호(진실원천)'를 최우선으로 확인하도록 통제했습니다.
2. **불필요한 상담원 연결(-$20) 최소화 (우선순위 큐 기반 Clarify & 오타 교정)**: 정보가 누락되거나 오타(예: "8ㅛㅣ")가 발생했을 때 즉각 실패 처리하지 않고, '누락 정보 큐(Queue)'를 통해 챗봇이 자연스럽게 정정 및 추가 질문을 던져 자동 복구해 냅니다.
3. **비용 효율적인 하이브리드 설계 적용**: 단순 예/아니오 대답이나 명백한 위험 패턴 감지 시에는 LLM API 호출을 생략(Fast-path)하여 LLM 호출 비용과 응답 지연을 최소화한 상태에서 자동화율을 극대화했습니다.

## 2. 성공 지표 (Success Metrics)

성공적인 PoC인지 판단하기 위한 핵심 KPI를 아래와 같이 제안합니다.

| 핵심 KPI | 목표 수치 | 선정 근거 및 의미 | 측정 방법 |
| :--- | :---: | :--- | :--- |
| **안전 종결률**<br>*(Safe Resolution Rate)* | **≥ 70%** | 하드 실패 없이 챗봇이 안전하게 대화를 마친 비율입니다. 명시적인 의료 상담을 챗봇이 안전하게 거절(Reject)한 케이스도 성공적인 방어로 보아 긍정적인 지표에 포함했습니다. | `(성공 + 안전 거절 + 복구 완료) / 전체 세션 수` |
| **완전 자동화 성공률**<br>*(Full Automation Success Rate)* | **≥ 45%** | 상담원의 개입 없이 챗봇이 스스로 예약 접수, 변경, 취소, 조회를 확정한 비율입니다. +$10의 비용 절감을 가져오는 가장 직접적인 비즈니스 지표입니다. | `상담원 개입 없이 확정된 예약 Action 수 / 예약 의도를 가진 전체 세션 수` |
| **대화 복구 성공률**<br>*(Recoverable Clarify Completion Rate)* | **≥ 60%** | 고객이 날짜나 시간을 빼먹고 말했을 때, 비싼 상담원 연결(-$20)로 넘기지 않고 챗봇이 자연스럽게 추가 질문을 던져 예약을 성공적으로 마친 비율입니다. | `Clarify 단계 진입 후 최종적으로 자동화에 성공한 건수 / 전체 Clarify 진입 건수` |

## 3. 안전 및 제약 지표 (Regression / Constraint Metrics)

아래 지표들은 절대 타협할 수 없는 '안전 마지노선'입니다. 임계값을 단 하나라도 초과할 경우, **PoC 테스트 및 운영을 즉시 중단**하고 원인 분석에 들어가야 합니다.

| 제약 지표 | 임계값 | 중단 기준 및 선정 근거 | 측정 방법 |
| :--- | :---: | :--- | :--- |
| **의료 상담 오답률**<br>*(Unsafe Medical Answer Rate)* | **0.0%** | 챗봇이 임의로 증상을 진단하거나 약물을 추천하는 등 의료법을 위반할 소지가 있는 답변을 한 비율입니다. 고객 신뢰를 심각하게 훼손하므로 단 1건도 발생해선 안 됩니다. | 런타임 Safety Gate 로그 분석, Golden Eval 테스트셋 검증 |
| **치명적 실패율**<br>*(Hard Fail Rate)* | **< 1.0%** | DB 저장에 실패했음에도 예약이 확정되었다고 거짓으로 안내하거나, 점심시간 및 정원 초과(1시간 3명 제한) 룰을 어기고 예약을 강행한 비율입니다. 환자의 헛걸음을 유발하므로 매우 엄격히 통제합니다. | 시스템 에러 로그, 챗봇 응답과 실제 DB 예약 내역의 불일치율 모니터링 |
